# Smart Contract Summary & Q&A Assistant

## 📌 Project Overview
The **Smart Contract Summary & Q&A Assistant** is an advanced Retrieval-Augmented Generation (RAG) system designed to help users interact intelligently with legal and contract documents. It securely ingests files, performs semantic search, and provides highly grounded answers accompanied by concrete source citations to prevent hallucinations.

This project was developed as a Workshop Application Project aligned with the **NVIDIA DLI Course** and submitted as part of the **ITI training program**.

---

## 🏗️ System Architecture
The application follows a strictly modular architecture to ensure scalability, maintainability, and clean code principles. It avoids notebook-style deployment in favor of a robust production-ready pipeline.

### Modular Components Flow:
1. **Document Ingestion (`processor.py`)**: Seamlessly handles PDF and DOCX file loading. It utilizes `RecursiveCharacterTextSplitter` to robustly chunk the text, preserving the critical semantic context of legal clauses.
2. **Vector Store & Embeddings (`database.py`)**: Responsible for generating dense vector embeddings using OpenAI models and storing them into **FAISS**, providing an efficient local semantic index for rapid similarity searches.
3. **RAG Logic & Reasoning (`engine.py`)**: The core brain of the assistant. It manages the `ConversationalRetrievalChain`, ensuring strict guard-rails for safety and factuality by prompting the LLM exclusively with the retrieved context.
4. **User Interface (`app.py`)**: A modern, interactive web layer built with **Gradio**, featuring a clean dual-tab layout for intuitive file uploading and real-time chat.

---

## ✨ Key Features
- **Multi-Format Ingestion**: Native support for processing both `.pdf` and `.docx` legal contracts.
- **Robust RAG Pipeline**: Optimized text chunking combined with high-dimensional semantic similarity.
- **Source Citations**: Every answer generated by the assistant is rigorously backed by direct page-level citations from the uploaded document, ensuring grounding and zero hallucinations.
- **Conversation State Tracking**: Supports multi-turn dialogues with dynamic context memory, allowing users to ask follow-up questions organically.
- **Strict Guard-rails**: Enforced prompt templates restrict the assistant from fabricating information, safely rejecting out-of-context queries.

## 🚀 Installation & Setup

### Prerequisites
- Python 3.9+
- An active OpenAI API Key

### Step-by-Step Guide

**1. Clone or Navigate to the Directory:**
```bash
cd Smart_Contract_Assistant
```

**2. Create and Activate a Virtual Environment (Recommended):**
```bash
python -m venv .venv
# For Windows:
.venv\Scripts\activate
# For Linux/Mac:
source .venv/bin/activate
```

**3. Install Dependencies:**
```bash
pip install -r requirements.txt
```

**4. Set up Environment Variables:**
Create a `.env` file in the root directory (or use the empty one provided) and add your OpenAI API Key:
```text
OPENAI_API_KEY=your_openai_api_key_here
```

**5. Run the Application:**
```bash
python app.py
```
*The Gradio interface will launch locally (usually on `http://127.0.0.1:7860`).*
